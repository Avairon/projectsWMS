# Промпт для генерации практической части дипломного проекта
## Тема: «Разработка веб-системы управления проектами образовательной организации»

---

## КОНТЕКСТ ПРОЕКТА

Вы — эксперт по написанию дипломных работ в области разработки программного обеспечения. Ваша задача — сгенерировать практическую часть дипломного проекта объёмом 10-15 страниц текста (без учёта кода и схем) на основе предоставленной информации о реализованной системе.

### Описание системы

Разработана веб-система «Реестр проектов НХТК» для автоматизации управления проектами в образовательной организации (колледже). Система предназначена для координации работы над учебными и производственными проектами, контроля выполнения задач и формирования отчётности.

**Технологический стек:**
- Backend: Python 3.12+, Flask (веб-фреймворк)
- Frontend: HTML5, CSS3, JavaScript, Bootstrap 5, Flatpickr (календарь)
- База данных: JSON-файлы (users.json, projects.json, tasks.json, tokens.json, directions.json)
- Дополнительные библиотеки: Flask-Login, openpyxl (экспорт Excel), python-dateutil

**Архитектура системы:**
```
/workspace/
├── app/
│   ├── __init__.py          # Фабрика приложения Flask
│   ├── models.py            # Модель пользователя (UserMixin)
│   ├── tables.py            # Класс DataTable для фильтрации данных
│   ├── utils.py             # Утилиты (БД, токены, файлы, история)
│   ├── routes/
│   │   ├── auth.py          # Аутентификация и регистрация
│   │   ├── dashboard.py     # Главная панель и статистика
│   │   ├── projects.py      # CRUD проектов
│   │   ├── tasks.py         # CRUD задач и подзадач
│   │   └── reports.py       # Отчёты с фильтрацией и Excel
│   ├── templates/           # HTML-шаблоны (14 файлов)
│   └── static/              # Статические файлы
├── database/                # JSON-хранилище данных
├── uploads/                 # Загруженные файлы отчётов
├── config.py                # Конфигурация
└── run.py                   # Точка входа
```

**Ролевая модель:**
1. **Администратор** — полный доступ, управление пользователями, направлениями, генерация токенов
2. **Руководитель проекта (manager)** — создание проектов, назначение кураторов и исполнителей
3. **Куратор (supervisor)** — контроль выполнения задач проекта
4. **Исполнитель (worker)** — выполнение задач, загрузка отчётов

**Ключевой функционал:**
- Регистрация пользователей по токенам (система одноразовых токенов)
- CRUD операций для проектов, задач, подзадач
- Иерархия: Проект → Задачи → Подзадачи с дедлайнами
- Загрузка файлов отчётов (png, jpg, pdf, doc, xlsx и др.)
- История изменений задач (аудит действий пользователей)
- Фильтрация и поиск по таблицам (текстовый поиск, диапазон дат)
- Экспорт отчётов в Excel с форматированием
- Разграничение доступа на основе ролей и принадлежности к проектам
